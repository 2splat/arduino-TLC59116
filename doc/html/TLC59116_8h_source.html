30  ### The TLC59116 Device
31  See the TI spec sheet for much detail: http://www.ti.com/lit/ds/symlink/tlc59116.pdf
32 
33  - Output pins ("channels") are numbered 0..15.
34  - Channels are "sinks", acting like Arduino pins that are LOW, even for PWM.
35  - Connect the minus side of LEDs to the output pins.
36  - Channels are considered ON, or OFF, instead of HIGH/LOW (or, at some PWM value)
37  - PWM is 8 bit.
38  - I2C addresses start at 0x60. See TLC59116_Unmanaged for the available addresses (up to 14).
39  - This library discovers the devices on the I2C bus and keeps them in a list.
40  - Use the _index_ in this library: e.g. "0" is the device with the lowest address,
41  "1" is the next highest, etc.
42  - Individual devices cannot be "reset", all devices on the I2C bus are affected by a reset.
43  - By default, all devices respond to broadcast ("allcall") commands (a special address).
44  - This can be changed, or turned off, for each device (see TLC59116::allcall_address())
45  - Three additional "broadcast" addresses can be set. See TLC59116::SUBADR_address()
46  - Thus, a device can participate in 4 different broadcast groups (or none):
47  - allcall_address()
48  - SUBADR_address(0)
49  - SUBADR_address(1)
50  - SUBADR_address(2)
51  - See the various groupings of the functions (below) for functionality.
52 
53  ### Protocol:
54 
55  - Setting up
56  - Declare a static TLC59116Manager(static scope, or "static" keyword if you are very clever)
57  \snippet allfeatures/allfeatures.ino Need a manager
58  - Execute \ref TLC59116Manager::init() ".init()" the manager once (usually in setup())
59  \snippet allfeatures/allfeatures.ino Init the manager
60  - The devices should be in a ready state (by default, outputs are enabled, but all channels are off)
61  - Obtain a reference from yourmanager\[i], 0..device_count()-1
62  - This library passes and returns references so that "." can always be used.
63  \snippet allfeatures/allfeatures.ino Static referencing devices
64  \snippet allfeatures/allfeatures.ino Index referencing devices
65  - Get information about the device as needed from the \ref Info Functions
66  \snippet allfeatures/allfeatures.ino device info
67  - Set device \ref Global Functions, \ref Group/Broadcast Addresses, typically in setup()
68  \code{.cpp}
69  void setup() {
70  \endcode
71  \snippet allfeatures/allfeatures.ino device setup
72  - Initial \ref Digital Functions, \ref PWM Functions, typically in setup()
73  \snippet allfeatures/allfeatures.ino initial channel settings
74  - During loop, mess with \ref Global Functions, \re Group/Broadcast Addresses, \ref Digital Functions, \ref PWM Functions
75  - This library returns the device-reference for most calls, so you can chain functions
76  \snippet allfeatures/allfeatures.ino chain functions together

158  If you decrease the value (blink_time or brightness) in these functions,
159  the chip may act like the value is max for one "cycle".
160  This causes a full-brightness result for group_pwm, and a 10 second blink-length for group_blink.
